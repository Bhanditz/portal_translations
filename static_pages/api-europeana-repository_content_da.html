Europeana Repository

To organize its data Europeana uses a complex data model which is called Europeana Data Model (EDM). A detailed description of EDM is beyond the scope of this documentation: the reader is referred to the extensive documentation on Europeana website. However, a basic understanding of the data organization is necessary for implementing the API. The objective of this section is to provide this basic understanding.

Fields

Europeana data is organized in records which correspond to cultural heritage objects. Each record is described by a number of fields which contain such information as the title of the object, its creator, the place, persons and concepts related to it. TBD: Provide an Example.

Aggregated Fields

Europeana aggregates its data from cultural institutions that can use diverse, fine-grained systems and methodologies. As a result, a link between for example an object and a person may be stored in different specialized fields. To provide simpler views on this data, Europeana has introduced several general Aggregated Fields: title, who, what, when, and where. In these fields we gather together information from different record fields in order to make the discovery of objects easier.

Facets

The number of records that Europeana contains is very big and growing. Therefore we need efficient ways to allow our end users discovering what they need easily. One of such techniques is faceted indexing system that classifies each element along multiple dimensions. Besides being useful for searching data on Europeana portal, facets (those are those filters that you see on the left side of a portal page) can also be used by API users.
